Privacy Policy

For the Reebal Sami LinkedIn Data Portability Integration — a personal, local-only tool used by Reebal Sami to retrieve his own LinkedIn profile data.

Last updated: 23 April 2026

What this tool is

This document describes the privacy practices for a private, single-user software integration operated by Reebal Sami (https://reebal-sami.com). The integration uses LinkedIn's Data Portability (Member) API — informally the "DMA API" — to retrieve structured data about the operator's own LinkedIn account.

The integration is not a multi-user service, not an app installed on anyone else's device, and not a publicly available product. It is a developer tool used solely by Reebal Sami to keep his CV and cover-letter source data in sync with his LinkedIn profile.

Who this policy applies to

This policy applies only to data about Reebal Sami's own LinkedIn account — the account owner who registered the LinkedIn Developer application and authorized the OAuth flow.

The integration does not access, retrieve, or process data belonging to any other LinkedIn member, connection, message sender, or content author. No other person's data is ever collected by this tool.

What data is accessed

Through LinkedIn's Data Portability API, the integration may retrieve the following categories of data about the operator's own account only:

  • Profile information (name, headline, location, summary, contact info)
  • Work experience (positions, companies, dates, descriptions)
  • Education (institutions, degrees, fields of study, dates)
  • Certifications, licences, and publications
  • Languages, skills, and endorsements
  • Volunteer experience
  • Recommendations received

The integration does not retrieve: connection lists, private messages, InMail content, connection activity, feed data, ads interaction data, or any data about other LinkedIn members.

How data is used

Retrieved data is used to populate structured YAML files in Reebal's personal CV + cover-letter tooling repository (github.com/ReebalSami/portfolio-website). Specifically:

  • Refreshing the data layer behind config/cv/cv.public.yaml and config/cv/cv.private.yaml to keep the rendered CV in sync with LinkedIn
  • Populating cover-letter/profile/linkedin.md with the latest profile facts for use by Reebal's personal cover-letter drafting tool

The tool does not use retrieved data for: advertising, analytics, behavioral profiling, training AI models for sale, onward transfer to any third party, or any purpose outside Reebal's own career-documentation workflow.

Where data is stored

All retrieved data is stored locally on Reebal's personal development machine. Nothing retrieved through the LinkedIn Data Portability API is uploaded to any cloud service, deployed to any public website, or committed to any public git repository.

Git-hygiene: the directories that receive LinkedIn-retrieved data (cover-letter/ and config/cv/cv.private.yaml) are explicitly .gitignored so the content cannot be accidentally pushed to a public repository.

Data sharing

No data is shared with any third party. The tool is a one-way pull from LinkedIn into Reebal's local filesystem. There is no outbound data flow, no analytics provider, no crash reporter, no telemetry.

The only network calls made by the tool are (a) OAuth token exchange with auth.linkedin.com, and (b) authenticated GET requests to the LinkedIn REST API at api.linkedin.com.

Data retention

Retrieved data is kept on disk for as long as Reebal finds it useful for CV and cover-letter authoring. Retrieval is idempotent: re-running the sync replaces existing local files with fresh data. There is no accumulating history of retrievals.

If the tool is retired, the operator will delete the local data files and revoke the LinkedIn application's access token via LinkedIn's account settings.

Your rights (as the data subject)

Since the only data subject is Reebal Sami himself — the sole operator and authorizing account — standard GDPR subject rights (access, rectification, deletion, portability) are self-served at his discretion.

Any other LinkedIn member who believes this integration may have improperly accessed their data can contact Reebal directly (see below) — however, as detailed above, the integration by design does not retrieve data about any account other than Reebal's own.

Security

  • OAuth 2.0 authorization code flow with PKCE
  • Access tokens stored only in a local, .gitignored environment variable file
  • No access token or client secret committed to version control
  • HTTPS-only connections to all LinkedIn endpoints
  • Tool runs on Reebal's personal development machine with full-disk encryption (FileVault)

Changes to this policy

If the scope of data accessed changes, or if the tool's architecture changes in a way affecting privacy (e.g., hypothetically introducing a server-side component), this policy will be updated and the "Last updated" date above will reflect the change.
