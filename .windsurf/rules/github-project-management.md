---
description: Use GitHub Issues, Milestones, and Project board for all work tracking
---

- Every feature/skill MUST have a GitHub Issue before work begins.
- Every Issue must be assigned to a Milestone (M1-M5).
- Move Issues on the Project board: Backlog → In Progress → In Review → Done.
- When starting an Issue: move to "In Progress", create a feature branch (`feat/skill-NN-<name>`).
- NEVER commit or push before showing a local live demo to Reebal and getting approval.
- After approval: commit → push → close Issue → move to "Done".
- Use labels: `skill`, `bug`, `enhancement`, `reebal-action`, `cascade-action`, `blocked`.
- Assign Issues: implementation tasks to Cascade, account/approval tasks to @ReebalSami.
- Use `gh` CLI for Issue and Project management when possible.
