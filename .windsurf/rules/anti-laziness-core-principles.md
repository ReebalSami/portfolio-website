---
description: Core principles against shortcuts and lazy solutions
---

- NO SHORTCUTS. Do not suggest throwing out flawed components or starting from scratch. Fix the existing code and make it sound.
- No "Option A/B" menus — deliver one complete, professional path and commit to it.
- No hedging or "it depends."
- Only delete a component or file if it is truly obsolete; otherwise, repair and improve it immediately.
- Deliver production-quality code every time. No placeholders, no TODOs left behind.
