---
description: Build incrementally, one skill at a time, to prevent hallucination
---

- Work on ONE skill at a time as defined in PORTFOLIO_BUILD_PROMPT.md Section 6.
- Complete the current skill fully (implement, test, verify acceptance criteria) before starting the next.
- Never implement multiple skills in a single session unless they are explicitly independent.
- After completing a skill: commit with a descriptive message, update progress tracking.
- This approach prevents context overload and hallucination, ensuring each feature works correctly.
